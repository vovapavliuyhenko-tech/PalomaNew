# Оплата PayKeeper (Альфа-Банк) — Cloud Function

Сайт статический (GitHub Pages), поэтому логин/пароль от ЛК PayKeeper и
секретное слово хранить в нём нельзя. Они живут только здесь — в переменных
окружения функции в Яндекс Облаке.

```
checkout.html ──POST ?a=create──►  Cloud Function ──Basic auth──►  PayKeeper
      ▲                                                               │
      └────────────── редирект на ссылку оплаты ◄──────────────────────┘

PayKeeper ──POST ?a=webhook──► Cloud Function   (POST-оповещения в ЛК)
PayKeeper ──редирект─────────► thank-you.html?paid=1
```

Сервер PayKeeper: `https://paloma.server.paykeeper.ru`
ЛК: `https://paloma.server.paykeeper.ru/login` (логин `admin`)

## Что загружать в функцию

Перед каждой загрузкой выполнить в корне репозитория:

```bash
node paykeeper/sync.js
```

Он копирует сюда `paloma-products.js` и `coffee-menu-data.js` — по ним функция
пересчитывает цены. **После любого изменения цен на сайте — запустить снова и
перезалить функцию**, иначе сервер будет считать по старому прайсу.

В функцию загружаются: `index.js`, `package.json`, `paloma-products.js`,
`coffee-menu-data.js` (файл `test.js` на сервере не нужен).

## Настройки функции

| Параметр | Значение |
|---|---|
| Среда выполнения | `nodejs18` |
| Точка входа | `index.handler` |
| Таймаут | 10 секунд |
| Память | 128 МБ |
| Доступ | публичная функция |

Переменные окружения:

| Имя | Значение |
|---|---|
| `PK_USER` | логин ЛК PayKeeper (`admin`) |
| `PK_PASSWORD` | пароль ЛК PayKeeper (**секрет**) |
| `PK_SECRET` | секретное слово из ЛК → Настройки → POST-оповещения (**секрет**) |
| `PK_SERVER` | `https://paloma.server.paykeeper.ru` |
| `PK_SITE` | `https://paloma.website` |

## Настройки в ЛК PayKeeper

ЛК → **Настройки** → **Получение информации о платежах**:

- **Способ получения уведомления о платежах** → **POST-оповещения**
- **URL, на который будут отправляться POST-запросы:**
  ```
  https://functions.yandexcloud.net/<id-функции>?a=webhook
  ```
- **Секретное слово** — задать любое, и то же значение положить в `PK_SECRET`.
- **URL страницы, на которую клиент переходит при успешной оплате:**
  ```
  https://paloma.website/thank-you.html?paid=1
  ```
- **URL при неуспешной оплате:**
  ```
  https://paloma.website/checkout.html?payment=failed
  ```

## Подключение сайта

В `payment-config.js` в поле `PAYMENT_ENDPOINT` вписать адрес функции:

```js
PAYMENT_ENDPOINT: "https://functions.yandexcloud.net/<id-функции>",
```

Пока поле пустое — онлайн-оплата выключена, чекаут работает как раньше (заявка менеджеру).

## Защита цен

Сервер не верит ценам из браузера: они лежат в localStorage и их можно подменить.
`verifyCart()` сверяет каждую позицию с каталогом:

- букеты (`c1-…`) — от базовой цены до «база + максимальный размер + все допы»;
- кофе и десерты (`menu-…`) — по ценам из меню + максимум добавок;
- апселлы чекаута — точная цена;
- доставка — только 0 или 350 ₽;
- сертификаты/подписка/копилка — свободная сумма, но в границах 100…300 000 ₽.

POST-оповещение об оплате проверяется по подписи:
`key = md5(id + sum + clientid + orderid + секретное слово)`.
Ответ функции: `OK md5(id + секретное слово)`. Поддельные оповещения отбрасываются.

Проверить локально:

```bash
node paykeeper/test.js
```

## Тестовый → боевой режим

Кабинет пока в тестовом режиме. После успешной тестовой оплаты перевод в боевой
делает техподдержка PayKeeper (написать им, что тест прошёл).

## Касса (54-ФЗ)

Фискализация подключается на стороне Альфа/PayKeeper отдельно. Сайт уже передаёт
состав заказа (`service_name`) — этого достаточно для чека. Детали подключения
кассы уточняются у менеджера Альфа-Банка.
