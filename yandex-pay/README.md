# Оплата Яндекс Пэй — Cloud Function

Сайт статический (GitHub Pages), поэтому секретный ключ Яндекс Пэй хранить в нём нельзя.
Ключ живёт только здесь — в переменных окружения функции в Яндекс Облаке.

```
checkout.html  ──POST ?a=create──►  Cloud Function ──Api-Key──►  Яндекс Пэй
      ▲                                                              │
      └───────────── редирект на paymentUrl ◄────────────────────────┘

Яндекс Пэй ──POST ?a=webhook──► Cloud Function   (Callback URL в кабинете)
Яндекс Пэй ──редирект────────► thank-you.html?orderId=…&paid=1
```

## Что загружать в функцию

Перед каждой загрузкой выполнить в корне репозитория:

```bash
node yandex-pay/sync.js
```

Он копирует сюда `paloma-products.js` и `coffee-menu-data.js` — по ним функция
пересчитывает цены. **После любого изменения цен на сайте — запустить снова и
перезалить функцию**, иначе сервер будет считать по старому прайсу.

В функцию загружается вся папка `yandex-pay/` (кроме `test.js`, он не нужен на сервере).

## Настройки функции

| Параметр | Значение |
|---|---|
| Среда выполнения | `nodejs18` |
| Точка входа | `index.handler` |
| Таймаут | 10 секунд |
| Память | 128 МБ |
| Доступ | публичная функция |

Переменные окружения:

| Имя | Значение |
|---|---|
| `YPAY_API_KEY` | ключ из кабинета Яндекс Пэй (**секрет**, нигде больше не хранить) |
| `YPAY_MERCHANT_ID` | `71f5bcc2-e0e2-4b81-af9d-fce2791ac84d` |
| `YPAY_ENV` | `sandbox` для тестов, `prod` для боевого режима |
| `YPAY_SITE` | `https://paloma.website` |
| `YPAY_TAX` | код НДС по ФФД, по умолчанию `6` («без НДС», УСН) — **подтвердить у бухгалтера** |

## Подключение сайта

В `payment-config.js` в поле `PAYMENT_ENDPOINT` вписать адрес функции:

```js
PAYMENT_ENDPOINT: "https://functions.yandexcloud.net/<id-функции>",
```

Пока поле пустое — онлайн-оплата выключена, чекаут работает как раньше (заявка менеджеру).

## Callback URL в кабинете Яндекс Пэй

```
https://functions.yandexcloud.net/<id-функции>?a=webhook
```

Функция проверяет подпись вебхука (JWT, ES256, ключи с `/api/jwks`) и merchantId —
поддельные уведомления отбрасываются.

## Защита цен

Сервер не верит ценам из браузера: они лежат в localStorage и их можно подменить.
`verifyCart()` сверяет каждую позицию с каталогом:

- букеты (`c1-…`) — от базовой цены до «база + максимальный размер + все допы»;
- кофе и десерты (`menu-…`) — по ценам из меню + максимум добавок;
- апселлы чекаута — точная цена;
- доставка — только 0 или 350 ₽;
- сертификаты/подписка/копилка — свободная сумма, но в границах 100…300 000 ₽.

Проверить локально:

```bash
node yandex-pay/test.js
```
